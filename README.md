# GhostMusic 翻譯貢獻指南

歡迎來到 **GhostMusic** 機器人的多國語言（Locales）官方倉庫！這份 README 用來說明如何為 GhostMusic 貢獻翻譯檔案。

> ⚠️ **重要提醒**：本倉庫僅用於管理與存放翻譯文本。若您想對機器人的主程式碼進行變更或回報 Bug，請前往 [Ghost-Team 主專案倉庫] (<-這裡可以放你們主專案的連結)。

## 你可以貢獻什麼

- 新增全新的語言檔（例如：日文、韓文等）
- 補齊既有語言檔缺少的字串
- 修正翻譯內容、錯字、標點符號與格式

所有翻譯檔都集中放在 `locales/` 目錄下。

## 語言檔格式

每個語言檔都是一個標準的 JSON 檔案，例如：

- `locales/en.json` (英文基準檔)
- `locales/zh_TW.json` (繁體中文)

參與翻譯時請務必遵守以下規則：

- **檔名規範**：請使用標準語言代碼，例如 `ja.json`、`ko.json`、`zh_TW.json`。
- **編碼格式**：所有檔案必須使用 **UTF-8** 編碼。
- **JSON 語法**：必須維持正確的 JSON 格式，不能有多餘的逗號（Trailing commas）或重複的鍵名（Keys）。
- **結構一致**：鍵名（Key）結構必須與 `en.json` 完全一致。
- **保留佔位符**：請務必保留變數佔位符，例如 `{name}`、`{count}`、`{title}`，切勿更動或翻譯括號內的文字。

## 新增或修改翻譯的步驟

1. **Fork** 本專案到你的個人 GitHub 帳號。
2. 在 `locales/` 目錄下新增或編輯對應的語言 JSON 檔。
3. 在本地確認 JSON 格式可以正常解析。
4. 提交（Commit）變更並推送到你的 Fork 分支。
5. 發起 **Pull Request (PR)** 到本專案的 `main` 分支。

## 建議的送出前檢查

在您送出 Pull Request 之前，請再次確認以下事項：

- **範圍正確**：您的 PR 只修改了 `locales/` 底下的內容以及本說明檔，沒有夾帶其他無關檔案。
- **無遺漏**：對照 `en.json`，沒有遺漏必要的翻譯鍵（Keys）。
- **機密安全**：**絕對沒有**把任何 Discord Token、密碼或其他敏感私鑰提交進去。
- **佔位符完好**：所有的 `{變數}` 格式皆正確，沒有被不小心改壞。
